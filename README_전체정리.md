# 마비노기 모바일 파밍 자동화 (MabinogiAuto) — 전체 정리본

> 2026-07-18 기준 · 어비스 / 던전 / 사냥터 3개 콘텐츠 지원

**한 줄 요약** — 게임 화면을 OCR(글자 인식)로 읽어 어비스/던전/사냥터를 자동으로 **입장 → 클리어 대기 → 보상 수령 → 재입장 반복**하는 자동화.

---

## 1. 빠른 시작

1. 마비노기 모바일 실행 (**창 모드** 권장. 전체화면은 감지가 어긋날 수 있음)
2. 콘텐츠에 맞는 시작 화면으로 이동:
   | 콘텐츠 | 시작 화면 |
   |---|---|
   | 어비스 | 어비스 던전 선택 화면 |
   | 던전 | 원하는 던전의 구역 선택 화면 (또는 진입 옵션/결과 화면) |
   | 사냥터 | 원하는 사냥터의 첫 화면 (하단에 '파티 찾기 / 입장하기') |
3. `MabinogiAuto.exe` 더블클릭 → 관리자 권한(UAC) "예"
4. 컨트롤 패널에서 콘텐츠 선택 + 상세 설정 + 반복 방식(무한/횟수/시간) 지정
5. **[시작]** 클릭

### 버튼 / 단축키

| 버튼 | 동작 |
|---|---|
| 시작 (F9) | 반복 시작 |
| 안전 중지 (F9) | 진행 중인 회차를 끝까지 마치고 멈춤 (다시 누르면 예약 취소) |
| 즉시 중지 (F10) | 바로 종료 (게임은 당시 화면에 그대로 남음) |
| 어비스/던전/사냥터 설명서 | 선택한 카테고리의 사용법 팝업 |
| 적용된 설정 | 켜 둔 설정 + 자동 안전장치 목록 팝업 |
| 권장 창 모드 | 게임 창을 OCR 최적 크기(1272 또는 1908)로 즉시 변경 |
| 설정 저장 | 설정만 저장 (다음 회차부터 적용) |

> F9/F10은 게임 화면에 포커스가 있어도 동작합니다.

**참고**
- 관리자 승인(UAC)은 exe를 켤 때 한 번만 필요합니다.
- 로그 창에 회차 진행과 각 단계가 색상 구분되어 실시간 표시됩니다.
- 자동화는 각 버튼/키를 필요한 만큼만 누르며, 전투 조작은 하지 않습니다 (전투는 게임 내 자동전투가 수행).

## 2. 파일 구성

| 파일 | 설명 |
|---|---|
| `MabinogiAuto.exe` | **시작점.** 더블클릭으로 실행 (GUI 컨트롤 패널). 단일 파일이라 다른 PC엔 이것만 복사하면 됨 |
| `mabinogi_gui.ps1` | 컨트롤 패널 원본 (exe에 내장됨) |
| `mabinogi_run_once.ps1` | 1회차 실행 본체 — 클릭·OCR·키 입력 (exe에 내장됨) |
| `config.json` | 설정 파일 (좌표·대기시간·키 등, 한국어 설명 포함) |
| `rdp_redirect_console.ps1` | RDP 자동 전환이 실행하는 내부 스크립트 (삭제 금지) |
| `Log\` | 실행 로그 + 오류 세트 (`error_시각.png` / `error_시각.log`) |
| `이슈_개선점_목록.md` | 알려진 이슈 / 확인 필요 항목 / 개선 아이디어 기록 |
| `build\` | exe 빌드 스크립트 (개발용 — 실행에는 불필요) |
| `mabinogi_controller.ps1` | [레거시] 구버전 콘솔 컨트롤러. exe에 포함되지 않음. GUI와 동시 실행 금지 |

- exe는 실행 시 스크립트를 `%LOCALAPPDATA%\MabinogiAuto`로 추출해 실행합니다. `config.json`도 그 폴더의 것을 사용합니다 (설정은 GUI에서 바꾸는 것을 권장).
- exe 업데이트 시 좌표/구조는 자동으로 최신화되고 사용자 설정은 유지됩니다 (coordsVersion 마이그레이션).
- RDP 자동 전환은 시작/설정 저장 시 자동으로 설치됩니다. 끄려면 `config.json`의 `rdp.autoConsoleRedirect`를 `false`로 바꾸고 [설정 저장]을 누르면 예약 작업이 제거됩니다.

## 3. 콘텐츠별 동작 흐름

### 어비스 (허상의 정박지 / 광기의 동굴 / 흩어진 물길)

1. 던전 카드 클릭 → 상세 화면 확인 → 혼자하기/함께하기 탭 → 난이도 클릭(설정 시)
2. 입장 — 함께하기는 매칭 방식 4종 지원:
   | 매칭 | 동작 |
   |---|---|
   | 우연한 만남 | 토글 켜고 입장하기 → 인원이 모이면 자동 입장 |
   | 파티 찾기 | 토글 끄고 파티 찾기 → 매칭 완료 시 자동 입장 |
   | 파티(파티장) | 직접 짠 파티 그대로 입장하기 (채우지 않음, 인원 부족 시 도전 확인 자동) → 전원 준비되면 자동 입장 |
   | 파티(파티원) | 메뉴 이동 없이 필드 대기 → 파티장이 입장 시작하면 '준비 완료' 자동 클릭 → 따라 입장 |
3. 입장 감지 → 자동출발/음식 키 → 클리어 대기 → 화면 터치 → 나가기 → ESC → 어비스 → 선택 화면 복귀 → 반복
   - 캐릭터가 멀면 '이동하기' 자동 처리 → 도착 후 입장
   - 파티(파티원)은 선택 화면 복귀 없이 필드에서 다음 입장을 대기

### 던전

- 선택/옵션/결과 화면 자동 인식 → 난이도(일반/어려움) + 스테이지(1-1~2-3)
- 진입 옵션: 은동전 소탕/더블 루팅 카드 설정 (소진 시 단계적 폴백 옵션)
- 우연한 만남 토글 또는 파티찾기 → 입장 → 클리어 대기 → 화면 터치 → 엔딩 컷신 넘김 → 결과 화면 → **다시 하기** → 옵션 화면 복귀 → 반복
- 은동전 소진 + '소진 시 계속' 옵션 꺼짐 = 나가기 후 정상 정지(파란 안내)

### 사냥터

- 원하는 사냥터 첫 화면에서 시작 → 난이도(일반/어려움/매우 어려움) 선택
- 은동전(사냥 임무) / 더블 루팅 카드 설정 + 소모량(10/20) 교차 검증
- 파티찾기 또는 바로 입장('입장하기'/'임무 시작') → 사냥 완료 대기 → 화면 터치 → 컷신/전리품 공개 → 결과 화면 → **새 임무 선택** → 첫 화면 복귀 → 반복
- 특정 사냥터에 매이지 않는 범용 방식. 가방이 차면 아이템 정리 화면을 자동으로 넘김 (게임 내 정리 규칙대로 분해/판매되니 게임의 정리 설정을 확인해 둘 것)
- 은동전 소진 정책: 잔량 10~19개 + 더블 루팅은 '소탕만 계속' 옵션에 따라 소탕으로 계속, **10개 미만이면 사냥터에서 나가고 자동 종료**

> 시작 시 게임이 어느 화면에 있든(상세/던전 안/클리어/보상/필드) 현재 상태를 인식해 알맞은 단계부터 이어갑니다.
> 화면 복귀만 수행한 실행은 '준비 실행'으로 회차에 세지 않습니다.

## 4. 내장된 안전장치 (GUI의 '적용된 설정' 팝업에서도 확인 가능)

- **자동 부활**: 행동불능 감지 → 남은 횟수 있으면 R(여기서 부활), 없거나 재료 부족이면 '여신상에서 부활' 클릭 후 전투 재개 (컨트롤 패널 체크로 켜고 끔)
- **자동사냥 꺼짐 감시**: 부활/이동 후 자동사냥이 꺼져 있으면 Space 재입력 (나침반 아이콘 픽셀 판별 — 전투 중 스킬 버튼일 때는 누르지 않음)
- **보스/엔딩 컷신** '장면 넘기기' 자동 클릭 (클릭 직전 재확인으로 오클릭 방지)
- **구매 팝업**(회복 물약 부족 등) 자동 닫기
- **아침 6시 리셋 체인** 자동 넘기기: NPC 대화 → 출석부 건너뛰기 → 출석 완료 확인(Space) → 오늘의 스텔라 픽(카드 선택→확정) → 공지/이벤트 팝업까지 전부 자동
- **던전 입장 확인 팝업**: '일주일 동안 보지 않기' 체크 후 입장
- **전투가 긴 판 보호**: 클리어 대기 한도를 넘겨도 전투가 진행 중이면(퀘스트 추적기 확인) 자동 연장 (설정값의 3배까지)
- **DPI/창 크기 변화**: 비율로 자동 보정 (16:9 창모드 권장)
- **화면 캡처 실패**(RDP 최소화 등): 오류로 죽지 않고 "일시정지", 복구되면 이어서 진행. 그동안 제한시간도 멈춤. 로그에 원인(RDP 끊김/최소화/화면 꺼짐)을 구분해 안내
- **자동 전면화**: 게임 창이 가려져 감지가 계속 실패하면 자동으로 앞에 가져옴 (사용자가 PC 조작 중이면 안 뺏음)
- **클릭 재시도**: 클릭이 빗나가면 확인 후 다시 클릭 (취소류 버튼은 상태를 확인해 오클릭 방지)
- **복귀 상태 머신**: 나가기 후 어떤 화면이든 판단해 선택 화면까지 복귀
- **중복 실행 방지**: 이미 돌고 있으면 두 번째 실행은 스스로 종료 (GUI에 안내)
- **오류 시 진단 덤프**: 실패 순간 스크린샷 + 로그 전문 사본을 같은 이름 세트(`error_시각.png` + `error_시각.log`)로 Log에 저장, OCR 원문 포함 (각 최근 20개 보관)

## 5. RDP(원격 데스크톱) 사용 시

| 상황 | 동작 |
|---|---|
| RDP 창 열어둠 | 정상 작동 |
| RDP 최소화 | 자동화 "일시정지" (다시 열면 이어감) — **자리 비울 땐 최소화 대신 창을 닫을 것!** |
| RDP 닫기/끊기 | 자동으로 본체 화면 전환 → 계속 작동 (전환 기록: `Log\rdp_redirect.log`) |

- 주의 1: RDP를 끊으면 본체 모니터에 바탕화면이 잠금 해제된 채 표시됩니다. 안전한 장소의 PC에서만 사용하세요.
- 주의 2: 화면 전환 시 게임이 전체화면으로 바뀔 수 있습니다. 게임 설정에서 "창 모드"를 명시 지정해두면 예방됩니다.
- 참고: "최소화"해도 계속 돌리고 싶으면, 접속하는 쪽 PC(클라이언트)의 관리자 PowerShell에서 아래 한 줄 실행 후 RDP 재접속:

```powershell
New-ItemProperty -Path 'HKCU:\Software\Microsoft\Terminal Server Client' -Name 'RemoteDesktop_SuppressWhenMinimized' -Value 2 -PropertyType DWord -Force
```

## 6. 설정 변경 (config.json)

대부분의 설정은 GUI에서 바꾸고 [설정 저장]을 누르면 됩니다.
세부 좌표/시간은 메모장으로 `config.json`을 열어 수정 → 다음 회차부터 적용됩니다. (`_`로 시작하는 항목은 설명 주석)

| 항목 | 설명 |
|---|---|
| `timeoutsSeconds.dungeonClear` | 클리어 대기 시간 (기본 600초 = 10분, 전투 중이면 자동 연장) |
| `timeoutsSeconds.partyMatching` | 파티 매칭 대기 (기본 300초) |
| `afterEntry.keys` | 입장 후 누를 키와 로그 표시명 |
| `repeat.defaultCount` | 횟수 모드 기본값 |
| `focus.refocusEverySeconds` | 자동 전면화 주기 (0 = 끄기) |
| `focus.onlyWhenUserIdleSeconds` | 사용자 유휴 시에만 전면화 (0 = 항상) |
| `rdp.autoConsoleRedirect` | RDP 끊김 시 본체 전환 (`false` = 예약 작업 제거) |
| `diagnostics.keepScreenshots` | 오류 스크린샷 보관 개수 (기본 20) |

- `config.json`을 지우면 내장 기본값으로 실행됩니다 (고장 안 남).
- 좌표를 직접 수정해 쓰려면 config에 `coordsVersion`을 프로그램 값과 맞춰 적어야 유지됩니다 (낮으면 내장 최신 좌표로 대체됨).

## 7. 문제가 생기면

1. `Log\mabinogi_run_once.log` 마지막 줄 확인
   - `[안내/경고] 화면 캡처 실패 - RDP 창이 최소화...` → RDP 창을 다시 열기
   - `[안내/경고] 화면 캡처 실패 - RDP 연결이 끊긴 상태...` → 조치 불필요 (본체 화면 자동 전환 대기 중, 몇 초 내 자동 복구)
   - 위 둘은 **일시정지 상태일 뿐 오류가 아님**
   - `[오류]` + `[진단]` 줄 → 같이 저장된 `error_*.png`를 열어 실패 순간 게임 화면과 OCR 결과를 확인
   - "시작 준비 중 오류" → config.json 값 형식 오류 등 초기화 실패. 메시지의 원인을 확인하거나 config.json을 지우고 다시 시작
2. 흔한 원인
   - 게임이 전체화면(16:10)으로 바뀜 → 창 모드로 변경
   - 게임 창을 다른 창으로 가림 → 자동 전면화가 처리하나, 사용자가 계속 조작 중이면 전면화를 안 하므로 게임을 보이게
   - 시작 화면이 이상함 → 각 콘텐츠의 권장 시작 화면에서 시작 (1번 참고)
   - '다른 자동화 인스턴스가 이미 실행 중' → 작업 관리자에서 `powershell.exe` 종료 후 다시 시작
3. 로그는 회차 시작마다 새로 쓰지만, 오류가 나면 그 순간의 스크린샷과 로그 전문 사본이 같은 이름 세트로 Log 폴더에 남으므로 나중에라도 원인 확인이 가능합니다.
